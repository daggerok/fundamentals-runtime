# ТРЕБОВАНИЯ: SEC EDGAR Financial Fundamentals Dashboard

**Проект:** fundamentals  
**Версия:** 0.1 (Прототип)  
**Дата:** 2026-07-13  
**Назначение:** Этот документ фиксирует функциональные и нефункциональные требования к приложению. Предназначен для обсуждения проекта с AI-агентами, агентной разработки, форка или создания production-версии showcase.

---

## 1. Обзор / Видение

Лёгкое, самодостаточное веб-приложение, которое позволяет пользователям:
- Искать публичные компании США по тикеру или названию
- Загружать и отображать ключевые финансовые фундаментальные данные с использованием бесплатного SEC EDGAR XBRL API
- Изучать Отчёт о прибылях и убытках, Движение денежных средств, Баланс и Метрики на акцию
- Просматривать данные как в табличном виде, так и в виде простых графиков
- Переключаться между Годовым (Y) и Квартальным (Q) представлениями
- Настраивать отображение (тёмная тема, масштаб/зум)

**Основная цель (Прототип):** Предоставить рабочее single-file HTML-приложение, демонстрирующее использование данных SEC **без шага сборки** для конечного пользователя.

**Будущая цель (Showcase):** Полированная, поддерживаемая демонстрация, которая может служить референсом для инструментов исследования финансовых данных.

### 2.2 Маппинг данных и вычисления

- Поддержка нескольких имён US-GAAP концепций для каждой метрики (например, несколько возможных ключей для "Revenue")
- Вычисление производных метрик:
  - Gross Margin, Operating Margin, Net Margin
  - Free Cash Flow (OCF - CapEx)
  - ROE, ROA, Debt/Equity, Current Ratio
  - Revenue per Share, FCF per Share
- Фильтрация и дедупликация фактов по form (10-K / 10-Q) и периоду (FY / Qx)
- Корректная обработка отсутствующих значений (`—`)

### 2.3 Пользовательский интерфейс

- Адаптивный (desktop-first, мобильная версия usable)
- Тёмная / Светлая тема (сохраняется)
- Слайдер масштаба шрифтов (в заголовке каждой секции)
- Выпадающий список подсказок с навигацией с клавиатуры (↑↓, Enter, Esc)
- Состояния загрузки и сообщения об ошибках
- Липкая верхняя панель с поиском и управлением

### 2.4 Источники данных и прокси (текущие)

- `https://www.sec.gov/files/company_tickers.json` (через локальный прокси)
- `https://data.sec.gov/api/xbrl/companyfacts/CIK{...}.json` (через локальный прокси)
- Рекомендуемые локальные прокси:
  - `bunx local-cors-proxy --proxyUrl https://www.sec.gov --port 8011`
  - `bunx local-cors-proxy --proxyUrl https://data.sec.gov --port 8012`

### 2.5 За пределами текущего прототипа (Out of Scope)

- Учётные записи пользователей / аутентификация
- Исторические filings помимо CompanyFacts
- Сравнение нескольких компаний
- Экспорт (CSV, PDF, Excel)
- Графики сложнее простых inline SVG
- Фильтрация / поиск внутри результатов
- Оптимизированные для мобильных графики
- Оффлайн-режим (кроме базового кэша)
- Обновления данных в реальном времени
- Инвестиционные советы или дисклеймеры (пока минимальные)

## 3. Нефункциональные требования

### 3.1 Производительность

- Начальная загрузка страницы < 5 сек на хорошем соединении (сейчас тяжело из-за Babel)
- Время поиска + загрузки компании < 3 сек (без учёта прокси)
- Плавные взаимодействия (без лагов при смене масштаба или вида)
- Использование localStorage < 4MB на запись

### 3.2 Надёжность и соответствие требованиям

- Должен соблюдать лимиты SEC: **максимум 10 запросов/секунду на IP** (цель ≤ 8)
- **ОБЯЗАТЕЛЬНО** отправлять корректный заголовок `User-Agent` во всех запросах к SEC:
  - Формат: `ApplicationName contact@email.com`
- Должен корректно обрабатывать 403, 429, сетевые ошибки
- Не должен превышать разумный объём запросов в демо

### 3.3 Безопасность

- В коде не хранятся учётные данные
- Нет прямых вызовов из браузера к SEC без правильных заголовков (текущая проблема)
- CORS обрабатывается через локальный прокси (или будущий серверный)
- Нет выполнения недоверенного контента, кроме контролируемого Babel (признанный риск)

### 3.4 Поддерживаемость

- Сейчас: Один самодостаточный файл `index.html` (намеренно для прототипа)
- Код должен оставаться читаемым в одном файле на текущем этапе
- Вся конфигурация (маппинги метрик, цвета, порты прокси) должна быть легко находимой и изменяемой

### 3.5 Доступность и удобство использования

- Базовая поддержка клавиатуры (уже есть)
- Семантическая структура где возможно
- Чёткие подписи и единицы измерения
- Состояния ошибок, которые направляют пользователя (инструкции по прокси, повтор)

### 3.6 Портативность и развёртывание

- Работает при открытии напрямую (`file://`) с прокси (ограничено)
- Работает через `bunx serve` / любой статический сервер
- Развёртывается на GitHub Pages (требует рабочего решения с прокси для живого демо)
- Для конечных пользователей не требуются Node.js / инструменты сборки

## 4. Технические ограничения (текущие)

- **Один HTML-файл** со всем inline (React, Tailwind через CDN, Babel standalone)
- React 18 через UMD CDN
- `@babel/standalone` для транспиляции JSX в рантайме
- Tailwind через `https://cdn.tailwindcss.com`
- Нет внешнего шага сборки / бандлера для текущей версии
- Данные загружаются только на стороне клиента
- Основной язык — английский (метрики и UI)

**Известные намеренные компромиссы:**
- Runtime Babel (медленная загрузка, большой бандл)
- Нет TypeScript
- Нет тестов
- Монолитный код

## 5. Будущие / Showcase требования (при форке)

При создании переименованной showcase-версии рекомендуется учесть следующее:

| Категория | Требование |
|-----------|------------|
| Сборка | Использовать Vite / Parcel / esbuild. При желании можно выводить в один HTML через плагины |
| Backend/Proxy | Заменить local-cors-proxy на serverless-функцию или Cloudflare Worker |
| Данные | Добавить правильный User-Agent + rate limiter. Рассмотреть bulk-скачивание данных |
| Технологии | Добавить TypeScript, вынос компонентов, нормальное управление состоянием (Zustand / Jotai) |
| Функции | Сравнение нескольких тикеров, экспорт, лучшие графики (Recharts / Chart.js), исторические filings |
| UX | Полный аудит доступности, улучшенный мобильный опыт, онбординг |
| Качество | Unit + E2E тесты, CI/CD, линтинг, автоматические проверки безопасности |
| Юридические | Добавить заметную атрибуцию данных SEC + дисклеймер «Не является финансовой рекомендацией» |
| Производительность | Предзагрузка популярных тикеров или серверное кэширование |
| Развёртывание | GitHub Pages + живое демо с рабочим прокси, или Vercel/Netlify |

## 6. Метрики успеха (для будущих версий)

- Пользователи могут загрузить любой крупный тикер без ошибок
- Все основные метрики рендерятся минимум для топ-50 компаний S&P
- Загрузка страницы < 2 сек после удаления Babel
- Ноль 403 от SEC при правильной настройке
- Легко понимаемо и расширяемо другими разработчиками

## 7. Глоссарий

- **CIK**: Central Index Key (10-значный идентификатор для SEC-файлеров)
- **XBRL**: eXtensible Business Reporting Language
- **CompanyFacts**: Endpoint SEC, возвращающий все tagged финансовые факты компании
- **10-K / 10-Q**: Годовые / Квартальные отчёты
- **us-gaap**: Основное пространство имён таксономии в SEC-файлах
